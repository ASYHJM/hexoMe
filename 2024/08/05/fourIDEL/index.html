PyCharm

大一上学期要学Python，老师给我们安排的集成开发环境就是pycharm。它比起Python自带的IDEL，很是方便。比如你要写一个

如果是用Python自带的IDEL，那么这几个字，都须要手打，即一个字一个字地打出来。
但是有了pycharm，当你打出pr的时候，下方就会出现待选的print。然后一按Enter，print就出来了。

括号也是如此，按下左括号“(”，如果后方没有字，它就会自动帮你把右括号“)”补上。
pycharm分为专业版（professional）和免费的community版。直到昨天我都还以为老师给我们的pycharm是专业版。要不是我仔细看了一下图标，还真发现不了[捂脸]

>>pycharm的安装方法
目前为止，如果你是Python的初学者——甚至我都在用这个免费的community版pycharm。写Python足够用了。

VS Code

全称大概是“visio studio code”吧？我目前用markdown语言写这篇网页文档到时候就用的是VS code。感觉它算markdown编译器里“所见即所得”做得最棒的。其他家的markdown，代码层面，粗细都一样，除了颜色有分别，余下的功能都不咋地（个人观点）。
而VS code，代码里的一级标题，就是会加粗；Italic 就是斜的；中划线就是被划掉的，这些居然能够在代码层面就被显示。这个创意非常不错，给个五星好评！★★★★★

pycharm、sublime和notepad++虽然都能写markdown，但是视觉上绝对比不过VS code。所以我以后写markdown，可能都会用VS code。也许这就是“术业有专攻”在IDEL中的体现吧！
如此好用的markdown编译器，居然也是免费的 ! ! !
>>下载链接：https://code.visualstudio.com/insiders/

Sublime

这个就是收费软件了。但是Windows下，用“sublime”&“123云盘”作关键词，可以很轻松地找到“开心版 ”。我的sublime就属于这个版本。

如上图所示，我会用它来编写.yml文件。在配置我这个网站的主题时，我也用到了一部分的sublime（主要还是VS code——sublime是在几乎配置好主题的之后才下载的，所以好像没什么用武之地?）。
但sublime还是很简洁的。平常开一个pycharm，运存就花掉1000多MB（多的时候甚至达到2000 MB，比AutoCAD还吃运存），VS code也要花掉500多MB。但是sublime只要花费约50 MB，大概是前者十分之一的运存即可。我已经把.yml格式文件的默认打开方式设定为sublime了。

Notepad++

简洁，确实非常简洁。运存做得也很低。是4个编译器里最低的。notepad++的运存只要不到20 MB。
但目前还没找到它能干的活计。markdown我用VS code，Python那肯定用pycharm（终端做得不错，交互的功能是刚需）。sublime在yml里发挥的“Ctrl+F”功能比Notepad++要舒服。
反观Notepad++，好像都不行。要么是我查找能力有限吧，查不出解决方案。
等以后再说吧。身边也有同学用notepad++写代码的，有空问问是什么语言。
